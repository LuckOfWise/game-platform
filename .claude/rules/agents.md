# エージェント運用ガイド

## 利用可能なエージェント

| エージェント | 目的 | 使用タイミング |
|-------------|------|---------------|
| planner | 実装計画策定 | 複雑な機能、リファクタリング |
| backend | バックエンド開発 | モデル、コントローラー、マイグレーション |
| frontend | フロントエンド開発 | ビュー、CSS、JavaScript |
| reviewer | コードレビュー | コード変更後（必須） |
| security-reviewer | セキュリティ分析 | コミット前、認証関連変更時 |

## 即座にエージェントを使用するケース

ユーザー指示なしで使用：

1. **複雑な機能リクエスト** → **planner** エージェント
2. **コード変更完了後** → **reviewer** エージェント
3. **バックエンド実装** → **backend** エージェント
4. **フロントエンド実装** → **frontend** エージェント
5. **セキュリティ懸念** → **security-reviewer** エージェント

## 並列実行

独立した操作は並列で実行：

```markdown
# 良い例: 並列実行
3つのエージェントを同時に起動:
1. backend: モデル実装
2. frontend: ビュー実装
3. reviewer: 既存コードのレビュー

# 悪い例: 不要な逐次実行
agent1完了 → agent2実行 → agent3実行
```

## エージェント間連携

### backend → frontend
- コントローラー実装完了後
- APIエンドポイント追加後

### frontend → backend
- 新しいデータが必要な場合
- バリデーション追加が必要な場合

### 全エージェント → reviewer
- 実装完了後は必ずレビュー

## 作業フロー例

### 新機能実装
```
1. planner: 計画策定
2. backend: バックエンド実装（並列可能なら）
   frontend: フロントエンド実装
3. reviewer: コードレビュー
4. security-reviewer: セキュリティ確認（必要な場合）
```

### バグ修正
```
1. backend または frontend: 修正実装
2. reviewer: コードレビュー
```
